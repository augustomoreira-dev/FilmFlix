# Skill — Spec Driven Development (SDD)

## Objetivo
Garantir que toda implementação seja guiada por especificação antes do código.

---

## Regras Obrigatórias

- Nunca implementar antes da especificação.
- Nunca gerar módulos inteiros sem validação.
- Sempre trabalhar incrementalmente.
- Sempre validar arquitetura antes da implementação.
- Código deve seguir os ADRs definidos.
- Implementações devem respeitar separação de responsabilidades.

---

## Fluxo obrigatório

1. Especificação
2. Revisão
3. Arquitetura
4. Contratos
5. Implementação
6. Validação
7. Refatoração controlada

---

## O que evitar

- overengineering
- geração massiva de código
- acoplamento forte
- duplicação
- lógica espalhada
